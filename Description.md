the flow of this tool is: the Runner Script is the bridge and the entry point between the user's program and our tool, the main brain of our tool is the Autonomous Controller, it decide when to save checkpoint, when to load, and when to stop. the Autonomous Controller always checking if it's time for checkpoint by checking the trigger state that is continusly updated in the State Tracker by the State Provider, and if it's time the Autonomous Controller will ask the state tracker to send a sanpshot with all the updated values of the states that are tracked to save program's progress, and the state tracker will send to state provider to send him all the updates values, then the Autonomous Controller will send the snapshot to the checkpoint Manager to create the checkpoint files. And while resume, the Autonomous Controller will tell the Checkpoint Manager to load the latest checkpoint file and send a snapshot of the tracked values saved to the state tracker to send them to the State Provider to load them back to the user's program
